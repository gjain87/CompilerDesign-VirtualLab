AIM -

To implement a LL(1) Parser.

Theory -

LL(1) parser is a Top Down Parser.It works on Context Free Grammar which is free from Ambiguity , Un-determinicy ,Left-Recusrion. LL(1) Parser has the follwoing components -

  • Input Buffer
  • Parse Stack
  • Parse Table
In LL(1) Parser name -
  • First L signifies left to right sccaning of the input token
  • Second L signifies leftmost derivation of the Parse Tree
  • 1 signifes the number of lookahead symbol

Procedure -

LL(1) parser first crates a parsing table and then process the stream of token accroding to the follwoing rules-

  • Push the start symbol into the stack
  • Compare the top most symbol of the stack with the lookahead symbol
  • If matching occurs , then pop the grammar symbol from the stac and increment the i/p pointer.
  • Output the production which is used for expanding a non-terminal , i.e. the result is a production which is used for push operation.

For Example -

E --> TE'
E' --> +TE' | ε
T --> FT'
T' --> *FT' | ε
F --> id | (E)

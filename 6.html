AIM -

To check whether the given Identifier is valid or Not.

Theory -

In Computer Programming indetifiers are considerd as a lexical token that names the language's entities. Identifiers refers to name given to entities such as variables , functions , structures , etc. In a program different indetifiers refering to different entities must be unqiue.

Procedure -

There are certain pre-defined rules for naming convention of an identifier. These rules are as follows :

  • The first character of an identifier should be either an alphabet or an underscore, and then it can be followed by any of the character, digit, or underscore.
  • It should not begin with any numerical digit.
  • In identifiers, both uppercase and lowercase letters are distinct. Therefore, we can say that identifiers are case sensitive.
  • Commas or blank spaces cannot be specified within an identifier.
  • Keywords cannot be represented as an identifier.
  • The length of the identifiers should not be more than 31 characters.

Therefore we need to check the given rules to consider an indetifier as valid or not.
